Held Messages

When a policy rule triggers a hold, the message is paused here for your review before it can be sent. This is a safety mechanism — it lets you inspect messages that match sensitive rules before they leave the system.

Release sends the message back to the queue for delivery. Deny permanently blocks it. Both actions are recorded in the audit trail.
