Policy Rules

Rules control what agents can and cannot do. Every mail action (send, read, delete) is checked against these rules before it executes. MSNGR evaluates three policy layers in order:

Config (config.yaml) — Safety limits like max message size, max recipients, and forbidden file types. These are always enforced first and cannot be changed from the web UI.

Rules (this page) — The rules you create here. This is where most policy decisions happen.

Runtime — Temporary states like rate limits and one-time approvals.

How decisions work: Rules are checked in priority order (lowest number first). Each rule results in Allow (proceed), Deny (block with explanation), or Hold (pause for operator review). A deny stops evaluation immediately. The default outcome when no rule matches is deny (safe by default).

Match criteria is a JSON object that defines when a rule applies. Common fields: agent_id, domain, action_type (send/read/delete), keywords_present, max_recipients, attachment_present.

Field reference & examples

A JSON object where all specified fields must match (AND logic). Omitted fields match everything.

FieldTypeDescription
action_typestringAction being performed: "send", "read", "delete"
agent_idstringAgent name (display name). Exact match.
account_idnumberNumeric account ID
domainstringRecipient domain. Use /regex/ for patterns.
senderstringSender address. Use /regex/ for patterns.
recipientstringAny recipient (To or Cc). Use /regex/ for patterns.
keywords_presentarrayAll keywords must appear in subject + body (case-insensitive)
keywords_absentarrayNone of these keywords may appear in subject + body
attachment_presentbooleantrue = has attachments, false = no attachments
attachment_typesarrayMIME types or extensions: [".pdf", "image/png"]
max_sizenumberDeny if message size exceeds this (bytes)
max_recipientsnumberDeny if total recipients (To+Cc) exceeds this

String matching: Plain strings are case-insensitive exact matches. Wrap in /slashes/ for regex: "domain": "/(gmail|yahoo)\\.com/"

Examples:

{"action_type": "send", "domain": "sys.lu"} — match sends to sys.lu

{"keywords_present": ["CONFIDENTIAL"]} — match any message containing "CONFIDENTIAL"

{"action_type": "send", "max_recipients": 5} — match sends with more than 5 recipients

{"agent_id": "Claude", "attachment_present": true} — match when Claude sends attachments
